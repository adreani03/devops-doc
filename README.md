# 🛠️ DevOps & Python Best Practices Documentation

Este repositorio contiene una colección de guías paso a paso, tutoriales y notas técnicas sobre las mejores prácticas de **DevOps** enfocadas en proyectos de **Python**. Aquí encontrarás recursos para configurar desde el control de versiones hasta despliegues automatizados y contenedores.

---

## 📂 Estructura del Proyecto

El contenido está organizado en dos secciones principales:

### 🚀 Guías Paso a Paso (`/step-by-step`)

Tutoriales detallados para implementar herramientas y flujos de trabajo en tus proyectos:

* **[Docker & Poetry](./step-by-step/docker.md)**: Configuración de contenedores optimizados usando multi-stage builds.
* **[Project Docker](./step-by-step/project_docker.md)**: Implementación específica de Docker para el proyecto Lumen Bridge.
* **[GitHub Actions](./step-by-step/github_actions.md)**: Automatización de flujos de trabajo (CI/CD).
* **[MkDocs Documentation](./step-by-step/mkdocs.md)**: Cómo crear sitios de documentación estética y funcional.
* **[Pre-commit Hooks](./step-by-step/pre_commit.md)**: Automatización de linters y formateadores locales.
* **[Project Workflows](./step-by-step/project_workflows.md)**: Descripción de los flujos de CI/CD aplicados en entornos reales.
* **[To-Do System](./step-by-step/todo.md)**: Gestión de tareas y seguimientos dentro del desarrollo.

### 📝 Notas y Referencias (`/notes`)

Documentación de referencia rápida y configuraciones específicas:

* **[Pre-commit Notes](./notes/pre-commit.md)**: Referencias rápidas sobre la configuración de hooks.

## 🛠️ Tecnologías Cubiertas

| Herramienta | Uso Principal |
| :--- | :--- |

## 📖 Cómo usar este repositorio

1. **Navega** por las carpetas `notes` o `step-by-step`.
2. **Sigue las guías** para replicar las configuraciones en tus propios proyectos.
3. **Consulta las notas** cuando necesites un recordatorio rápido sobre una configuración específica.
